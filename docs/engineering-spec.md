# Go Home 工程说明

本文是 Go Home 当前版本的工程主约束。后续实现、接口和 UI 均以本文为准。

## 1. 角色划分

### 公网服务器

公网服务器是唯一的管理中心，负责：

- Web 控制台静态文件加载。
- WebSocket JSON-RPC 业务 API。
- 管理员登录与单点登录。
- 家庭创建、家庭可见性、私密家庭授权。
- 家庭服务器绑定和解绑。
- 设备黑名单、强制下线、在线状态和流量统计。
- P2P UDP 打洞信令协调。

公网服务器绝不转发隧道数据，不能解密 UDP 隧道数据，也不能作为中继使用。

### 家庭服务器

家庭服务器运行在家庭局域网内，负责：

- 连接公网服务器并保持信令 WebSocket。
- 上报家庭真实 LAN 网段变化。
- 代远程客户端向家庭路由器申请 DHCP 租约。
- 对远程客户端家庭侧代理 IP 执行 ARP 代理。
- 扫描家庭局域网设备，生成真实 IP 与虚拟映射 IP 对照表。
- 与客户端建立 UDP P2P 加密隧道。
- 对超出能力范围的广播、组播或二层数据进行过滤。

### 客户端

客户端负责：

- 输入服务器地址与授权码连接公网服务器。
- 读取自己可见的家庭列表。
- 选择家庭并连接该家庭唯一绑定的家庭服务器。
- 创建本地虚拟网卡或系统 VPN 配置。
- 在默认真实同网段模式或备用虚拟映射模式下访问家庭局域网。

客户端不能创建家庭，不能编辑家庭，不能绑定或解绑家庭服务器。

## 2. 家庭与权限模型

家庭由公网服务器 Web 控制台创建。

家庭可见性分为：

- 公开家庭：所有通过授权码接入且未被拉黑的客户端可见并可连接。
- 私密家庭：只有被 Web 控制台授权的客户端可见并可连接。

一个家庭只能绑定一台家庭服务器。替换家庭服务器时，管理员必须先在 Web 控制台解绑旧家庭服务器，再绑定新家庭服务器。

一个家庭允许多个客户端同时连接。每个客户端拥有独立隧道会话、虚拟 MAC、家庭侧代理 IP、流量统计和状态机。

## 3. 网络模型

### 默认真实同网段模式

客户端默认使用家庭真实网段 IP，目标是尽量表现得像同一个交换机中的设备。

流程：

1. 家庭服务器上线后检测家庭 LAN 网段，例如 `192.168.3.0/24`。
2. 客户端选择家庭后，在 UDP 打洞前获取该家庭 LAN 网段。
3. 客户端检测本地所有物理网卡、虚拟网卡和系统路由。
4. 如果本地不存在网段冲突，则进入真实同网段模式。
5. 家庭服务器用客户端虚拟 MAC 代为向家庭路由器申请 DHCP。
6. 客户端获得家庭侧代理 IP，例如 `192.168.3.200`。
7. 家庭服务器对 `192.168.3.200` 执行 ARP 代理，真实局域网设备访问该 IP 时，由家庭服务器接住并转发到对应 UDP 隧道。

### 备用虚拟映射模式

当客户端本地网段与家庭真实网段冲突时，启用备用虚拟映射模式。

第一阶段只支持 `/24` 映射。

示例：

- 家庭真实网段：`192.168.3.0/24`
- 客户端备用虚拟网段：`192.168.6.0/24`
- 家庭真实设备：`192.168.3.5`
- 客户端访问地址：`192.168.6.5`

映射规则：

- 客户端访问 `192.168.6.5`，家庭服务器映射到 `192.168.3.5`。
- 家庭局域网设备访问远程客户端时，仍访问客户端的家庭侧代理 IP，例如 `192.168.3.200`。
- 家庭服务器维护双向映射表：
  - 家庭设备真实 IP 到客户端虚拟 IP。
  - 客户端家庭侧代理 IP 到远程客户端隧道。

备用虚拟映射模式允许广播、组播和自动发现能力下降。家庭服务器必须扫描家庭设备并在客户端 UI 中展示“真实设备 IP -> 客户端可访问虚拟 IP”的映射表。

### 网段冲突处理

客户端必须在 UDP 打洞前检测网段冲突。

处理规则：

- 家庭真实网段不冲突：使用真实同网段模式。
- 家庭真实网段冲突：提示用户设置备用虚拟网段。
- 备用虚拟网段仍冲突：禁止连接。
- 家庭 LAN 网段变化：家庭服务器自动上报，公网服务器推送给相关客户端，客户端重新评估模式。

## 4. DHCP 与 ARP 代理

客户端虚拟 MAC 是远程客户端在家庭局域网中的身份。

家庭服务器必须：

- 使用客户端虚拟 MAC 代为发起 DHCP。
- 获得家庭真实网段 IP 后，将该 IP 与客户端隧道会话绑定。
- 对该 IP 进行 ARP 代理，允许真实局域网设备把回包发给家庭服务器。
- 禁止把远程客户端原始 ARP 请求直接广播到真实局域网。
- 对多个客户端的代理 IP 和虚拟 MAC 做严格隔离。

正常断开时，家庭服务器应尝试释放 DHCP 租约。异常掉线时，家庭服务器保留租约状态直到过期，并在本地标记为暂不可复用。

## 5. 纯 P2P UDP 打洞

Go Home 不提供中继，不采用 IPv6，不存在备用转发链路。

打洞失败是正常业务状态。服务器和客户端必须将其表现为明确失败原因，而不是异常崩溃。

打洞策略包括：

- NAT 类型探测。
- UPnP / NAT-PMP 端口映射尝试。
- UDP 同步打洞。
- 端口预测。
- 多候选地址尝试。

所有策略失败后，客户端显示“当前网络环境无法穿透，直连失败”。

## 6. 加密与身份

全链路采用国密体系设计。

设备身份：

- 每个设备首次启动生成本地长期 SM2 密钥对。
- Device ID 由设备公钥、随机种子和 SM3 摘要派生。
- 授权码只用于接入校验，不作为长期身份。

WebSocket 消息：

- JSON-RPC 每条敏感消息携带 `timestamp`、`nonce`、`seq` 和 `time_key`。
- `time_key` 使用共享密钥材料与当前时间窗口生成。
- 服务器允许时间窗口容错。
- 连续 3 次校验失败强制下线。

UDP 隧道：

- 握手材料经公网服务器转发，但最终 UDP 会话密钥只由客户端和家庭服务器持有。
- 隧道数据使用 SM4 对称加密。
- 使用 SM3 或基于 SM3 的摘要进行完整性校验。
- UDP 包必须包含 `session_id`、`seq`、`nonce` 和方向标记。
- 使用滑动窗口防重放。
- 会话密钥按时间或流量阈值轮换。

跨平台国密库可以不同，但协议输出必须一致。

## 7. WebSocket 与 UDP 状态

WebSocket 是信令和管理通道，UDP 是隧道数据通道。

WebSocket 断开后启动 30 秒宽限期：

- 宽限期内 UDP 隧道继续通信。
- WebSocket 重连且校验通过后，无缝恢复。
- 超过 30 秒未恢复，销毁隧道和虚拟网卡。

## 8. 移动端限制

移动端采用受限局域网模式。它只保证能力范围内的家庭网段访问和分流，不承诺完整二层透明能力。

移动端默认只把家庭相关路由导入系统 VPN，互联网默认流量继续走物理网络。

移动端不要求完整支持 ARP、mDNS、NetBIOS、局域网游戏发现或投屏自动发现。超出能力的数据由家庭服务器过滤或由设备扫描列表替代。

## 9. UI 权限边界

服务器 Web 控制台：

- 创建家庭。
- 设置公开/私密。
- 授权私密家庭客户端。
- 绑定或解绑家庭服务器。
- 拉黑、解除拉黑、强制下线设备。
- 修改管理员密码和授权码。
- 查看服务器操作日志。
- 家庭列表支持搜索、公开/私密筛选、家庭服务器在线状态筛选。
- 保存、提交、授权、解绑、拉黑等操作必须弹出通知。
- UI 数据通过 WebSocket 热事件自动更新，不提供手动刷新按钮作为主路径。

客户端 UI：

- 连接服务器。
- 查看可见家庭。
- 选择家庭连接。
- 查看家庭服务器在线状态。
- 处理真实同网段或备用虚拟网段配置。
- 查看设备映射表、流量、丢包率、隧道状态。
- 查看由公网服务器计算得到的连接延迟。
- 断开连接和检查更新。

## 10. GitHub 与 CI/CD

项目发布到三个独立 GitHub 仓库：

- 公网服务器仓库：`https://github.com/CCdeAIHUB/go-home-server.git`
- 家庭服务器仓库：`https://github.com/CCdeAIHUB/go-home-homeServer.git`
- 客户端仓库：`https://github.com/CCdeAIHUB/go-home-client.git`

本地工作区可以继续保留单仓库开发形态。每次修改完成并通过基础校验后，使用 `scripts/publish-repos.ps1 -Push` 将对应内容拆分同步到三个远程仓库。GitHub Actions 在 push 后自动执行编译校验。
